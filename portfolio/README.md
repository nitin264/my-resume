# Portfolio Website

A personal portfolio website built with React, Vite, Tailwind CSS, and Framer Motion. The app is designed to showcase experience, skills, projects, and contact details in a responsive, animated, dark/light theme UI.

## Key Features

- **React + Vite** application with fast local development and production builds.
- **Tailwind CSS** for utility-first styling, plus custom global CSS in `src/index.css`.
- **Dark/light theme toggle** with persistence in `localStorage`.
- **Animated page sections** using `framer-motion` for smooth reveals and hover effects.
- **Responsive navigation** with desktop links and mobile drawer menu.
- **Data-driven content** stored in `src/data/index.js` for easy content updates.
- **Contact form** that opens the default mail client via `mailto:`.
- **Vercel Analytics** integration using `@vercel/analytics/react`.

## Folder Structure

- `src/`
  - `App.jsx` — root app component, includes page sections and theme provider.
  - `main.jsx` — application entry point.
  - `index.css` — global styles, theme variables, custom utilities, and layout.
  - `context/ThemeContext.jsx` — theme context provider and hook used across the app.
  - `components/`
    - `Navbar.jsx` — sticky navigation, mobile drawer, theme toggle, and hire button.
    - `Footer.jsx` — social links, copyright, and tech note.
    - `Button.jsx` — reusable animated button component for links and actions.
  - `sections/`
    - `Hero.jsx` — landing section with name, role, tagline, and CTA buttons.
    - `About.jsx` — summary, stats, and highlight list.
    - `Skills.jsx` — skill category cards with animated, glass-style panels.
    - `Experience.jsx` — timeline and role cards with alternating layout.
    - `Projects.jsx` — featured projects grid with tech badges.
    - `Contact.jsx` — contact links and a simple mailto form.
  - `data/index.js` — all page content, navigation links, skills, experience, projects, and contact info.
  - `__tests__/` — test files for key sections.

## Data and Content

The app content is mostly data-driven from `src/data/index.js`, including:

- Hero name, title, tagline, email, and LinkedIn.
- About summary, highlights, and closing sentence.
- Skill categories and items.
- Experience timeline, milestones, and responsibilities.
- Projects with descriptions, technologies, and feature highlights.
- Contact links and email address.

This makes the portfolio easy to update without changing the core UI logic.

## Styling and Theme

- The app uses Tailwind CSS with custom theme tokens in `src/index.css`.
- Dark mode is the default theme, with `data-theme="light"` applied for light mode overrides.
- Custom utilities include:
  - `.gradient-text`
  - `.project-card`
  - `.tech-badge`
  - `.glow-blob`
  - `.section`, `.container`, and `.label`
- The theme toggle persists user preference using `localStorage`.

## Scripts

- `npm install` — install dependencies.
- `npm run dev` — start the development server.
- `npm run build` — build for production.
- `npm run preview` — preview the production build.
- `npm run lint` — run ESLint across the project.

## Deployment

This portfolio is ready for deployment on Vercel or a similar static host.

- The default build output is generated by `npm run build`.
- The app uses Vite, so Vercel can deploy it as a static site with the `build` command and `dist` directory.
- A typical Vercel workflow for this repository can use `main` as production, `staging` for preview deployments, and `dev` for development branches.
- Environment is entirely frontend, with no server API required for the site itself.

## Notes

- The app includes `@vercel/analytics/react` for analytics tracking.
- The contact form uses `mailto:` to open the default email client.
- The mobile navigation drawer and smooth scrolling come from custom React state and DOM scroll logic.
- Test files are available in `src/__tests__/` for the About, Footer, and Projects sections.

## How to Update Content

Most content can be changed directly in `src/data/index.js`:

- `navLinks` for section navigation.
- `hero` for name, title, email, and LinkedIn.
- `about`, `skills`, `experience`, `projects`, and `contact` for page content.
